# Security Policy

## Credentials

Do not report exposed API keys in a public issue. Revoke the credential first,
then contact the repository maintainer privately.

AI Book Batch Writer does not intentionally persist API keys. Keys entered in
the UI remain in process memory for the current session. Saved projects and
desktop preferences exclude credential fields.

## Reporting a Vulnerability

Open a private security advisory on GitHub with reproduction steps, affected
versions and the expected impact. Avoid including real user content or active
credentials.
